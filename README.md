# 🚀 Kodik-README-AI-COP


## 🧪 Текущий этап разработки и планы
Уже реализовано (на что потрачены кредиты)
- ✅ Полный цикл сканирования и сбора контекста (дерево, манифест, главный файл).
- ✅ Интеллектуальный stackDetector, определяющий язык, фреймворк и команды для 10+ стеков (Node.js, Python, Go, Rust, Java, PHP, Ruby, Dart/Flutter, C#, Swift и др.).
- ✅ Модульная архитектура, упрощающая поддержку и расширение.
- ✅ Интеграция с локальными ИИ-моделями через OpenAI-совместимый API.
- ✅ Генерация README с жёсткой структурой (описание, возможности, стек, быстрый старт, структура).
- ✅ Проверка работы на нескольких реальных проектах (включая большой репозиторий ollama/ollama) для валидации качества.

Планы на будущее (не реализовано)
- 📋 Интерактивный CLI-опрос — добавление 2–3 вопросов для уточнения целевой аудитории и бизнес-задач проекта, чтобы описание стало ещё более точным.
- 📄 Анализ верхнеуровневой документации — чтение User Stories, логов коммитов и PR для лучшего понимания контекста и ценности фич.
- 🎯 Улучшение качества генерации — настройка промптов и параметров модели (температура, топ-р) для более релевантных ответов.
- 🧪 Масштабное тестирование — проверка на нескольких десятках open-source проектов с разными стеками для статистической оценки качества.


## 💸 Как были использованы 250 кредитов (токенов)
В рамках хакатона Kodik Launchpad мне были выделены 250 токенов для тестирования и создания проекта. Все кредиты были полностью израсходованы на следующие задачи

Отладка системного и пользовательского промптов — требовалось множество итераций, чтобы добиться строгой структуры ответа (заголовок, ключевые возможности, стек, быстрый старт, структура проекта) и корректного поведения модели на русском языке. Каждая итерация включала отправку запроса с деревом файлов, манифестом и кодом главного файла для разных проектов.
