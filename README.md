# 🚀 kodik-readme-ai

## 📝 Описание

**Kodik-README-AI-COP** — это микросервис на **Node.js (JavaScript)**. Бизнес-ценность: демократизация и упрощение локального использования искусственного интеллекта (ИИ) для разработчиков и бизнеса. Проект ориентирован на конечных пользователей. Он предоставляет REST API и управление через CLI.

## ✨ Ключевые возможности

- ✨ Запуск моделей
- ✨ управление моделями
- ✨ REST API
- ✨ квантование моделей
- ✨ кастомизация через Modelfile
- ✨ кроссплатформенность
- ✨ аппаратное ускорение.

## 🚀 Использование

```bash
node src/index.js [путь_к_проекту] [опции]
```

### Основные флаги и опции

| Опция | Альтернатива | Описание |
|-------|--------------|----------|
| `--target` | `-t` | Путь к проекту (по умолчанию текущая папка) |
| `--non-interactive` | `-n` | Пропустить интерактивный опрос |
| `--ai` | - | Включить/выключить AI-генерацию (`--no-ai` для отключения) |
| `--model` | `-m` | Модель AI (например, `gpt-4o-mini`, `llama3.1`) |
| `--api-url` | - | URL API (OpenAI-совместимый) |
| `--api-key` | - | API-ключ |
| `--tone` | - | Тон описания: `technical`, `marketing`, `minimal` |
| `--language` | `-l` | Язык для перевода (например, `en`, `ru`) |
| `--no-translate` | - | Отключить финальный перевод |
| `--output` | `-o` | Папка для сохранения README.md |
| `--config` | `-c` | Путь к файлу конфигурации (JSON/YAML) |
| `--dry-run` | - | Показать результат в консоли без сохранения файла |
| `--validate` | - | Запустить AI-валидацию сгенерированного файла |
| `--help` | `-h` | Показать справку |
| `--version` | `-v` | Показать версию |

### Примеры

**Генерация с маркетинговым тоном на английском:**
```bash
node src/index.js . --tone marketing --language en
```

**Использование файла конфигурации:**
```bash
node src/index.js --config ./my-config.yaml
```

**Проверка результата без записи в файл:**
```bash
node src/index.js ./src --dry-run
```

### Конфигурация

Вы можете создать файл `kodik-config.yaml` и передавать его через `--config`:

```yaml
ai:
  model: "gpt-4o"
  enabled: true
content:
  tone: "marketing"
  language: "en"
  translateSections:
    - "Description"
    - "Features"
```

### Приоритет настроек

1. Аргументы командной строки (самый высокий)
2. Файл конфигурации
3. Переменные окружения (`.env`)
4. Значения по умолчанию (самый низкий)
