# 🚀 kodik-readme-ai

## 📝 Описание

Инструмент для автоматической генерации файла README.md для любого программного проекта. Анализирует содержимое репозитория (структуру, манифесты, код, Git-логи) и создаёт качественную документацию, используя либо встроенную локальную логику, либо большие языковые модели (LLM) через OpenAI-совместимые API.

## ✨ Ключевые возможности

- **Автоматическое сканирование проекта** — Сканирует структуру проекта, манифесты, код и Git-логи для сбора контекста.
- **Генерация README в двух режимах** — Поддерживает локальную генерацию и генерацию через LLM (OpenAI-совместимые API).
- **Интерактивный опрос** — Собирает бизнес-контекст через интерактивный CLI-опрос.
- **Перевод разделов** — Автоматический перевод разделов README на указанный язык.
- **Валидация и исправление** — Валидирует сгенерированный README и автоматически исправляет ошибки.
- **Плагины** — Поддержка плагинов для расширения функциональности генерации.
- **Веб-интерфейс** — Веб-интерфейс для загрузки проекта и генерации README.
- **CLI-интерфейс** — Командная строка для генерации README с различными опциями.
- **Бенчмаркинг** — Инструмент для бенчмаркинга качества генерации на эталонных проектах.

## 🛠️ Стек технологий

- **Язык:** Node.js (JavaScript), Node.js (TypeScript), Rust
- **Фреймворк:** axum, Express
- **Пакетный менеджер:** npm
- **Дополнительно:** axios, adm-zip, simple-git, ejs, marked, winston, jest

## 📦 Быстрый старт

### Требования

- Node.js >= 18

### Установка и запуск

```bash
# Установка зависимостей
npm install

# Запуск
npm start
```

## 📂 Структура проекта

```
Kodik-README-AI-COP/
├── __tests__/ — тесты (e2e, integration, unit)
├── .benchmark-temp/ — эталонные проекты для бенчмаркинга
├── src/ — исходный код (commands, context, core, generator, interfaces, output, scanner, utils, validation)
├── services/ — сервис генерации
├── routes/ — API-маршруты
├── public/ — веб-интерфейс
├── plugins/ — плагины
├── prompts/ — промпты для LLM
├── scripts/ — скрипты (benchmark, reportGenerator)
├── locales/ — локализация (en, ru)
├── logs/ — логи
├── app.js — точка входа веб-сервера
├── package.json — метаданные проекта
└── README.md — сгенерированный README
```
